# 🏝️ Stage 5+ 확장 기획서 — 위·아래 섬 & 직원 대량생산 시스템

> 원본 러프 메모를 정리하고, 게임 디자인/아트/기술 관점에서 디테일을 채우고 확장했습니다.
> (현재 게임의 데이터 주도 구조 — `STAGES.unlocks`, Island 피처 빌더, Worker 모듈 — 위에 그대로 얹는 방향)

---

## 1. 한눈에 보는 컨셉

- **스테이지 5에 도달하면 "땅이 열린다"** → 메인 섬의 **위/아래로 새 섬**이 펼쳐진다.
- 새 섬들은 **직접 캐는 곳이 아니라**, **직원(NPC)을 고용해 자동으로 대량 수확**하는 "생산 시설" 구역이다.
- 각 직원은 **고용(코인 지불)** 후, **레벨업으로 시간당 획득량(생산 속도)**을 올릴 수 있다.
- 생산된 자원은 인벤토리로 들어가고 → (진열 알바가) 선반에 채우고 → 손님이 산다. 즉 **기존 경제 루프의 "공급량"을 폭발적으로 늘리는 후반 콘텐츠**.

### 기존 시스템과의 관계
| 구분 | 기존 "수확 알바"(Worker.js) | 신규 "시설 직원" |
|---|---|---|
| 동작 | 맵을 돌아다니며 준비된 노드를 순회 수확 | **한 자리(시설)에 상주**, 모션만 반복하며 자동 생산 |
| 대상 | 모든 자원 1개씩 순환 | **시설별 단일 자원**을 대량 생산 |
| 성장 | 레벨=수확 티어/속도 | 레벨=**시간당 생산량** |
| 목적 | 초중반 보조 | **후반 대량 공급** |

> 두 시스템은 공존한다. 수확 알바는 "소량·범용", 시설 직원은 "대량·전문".

## 2. 레이아웃 (맵 확장)

자유 카메라(드래그 팬 + 핀치/휠 줌)가 이미 있으므로, 위/아래로 섬을 늘려도 손으로 탐험 가능.

```
                 ┌───────────────┐
                 │   위 섬 (UP)   │   ← 도토리/바나나 대량 생산
                 │  도토리 · 바나나 │
                 └──────┬────────┘
                        │ 다리
        ┌───────────────┴───────────────┐        ┌──────────────┐
        │          메인 섬 (상점)         │──다리──│ 가든 섬(우측)  │  (기존 5단계)
        └───────────────┬───────────────┘        └──────────────┘
                        │ 다리
                 ┌──────┴────────┐
                 │ 아래 섬 (DOWN) │   ← 물고기/조개 대량 생산 (아주 작음)
                 │  양식장 · 조개장 │
                 └───────────────┘
```

- **위 섬**: 메인 섬 기준 화면 위쪽(-z 방향)에 배치, 나무 다리로 연결.
- **아래 섬**: 화면 아래쪽(+z 방향)에 배치, "아주 작게", 다리로 연결.
- 기존 **가든 섬(우측, +x)**은 그대로 유지(꽃·꿀).
- 다리/연결은 기존 `_buildBridge` 패턴 재사용(방향만 +z/-z로).

## 3. 위 섬 — 🌰 도토리 & 🍌 바나나 대량 생산 농장

### 3-1. 구성 / 디자인
- **가운데(섬 안쪽)**: **도토리(오크) 나무** 군락.
- **바다 쪽(섬 바깥쪽 가장자리)**: **바나나(야자) 나무** 군락.
- 나무들은 **장식 + 생산 노드** 역할. 직접 탭 수확 불가, **시간당 자동 수확**.
- 생산될 때마다 해당 나무에서 아이템이 톡 튀어나와 → 인벤토리로 날아가는 연출.

### 3-2. 직원 (2명, 각각 구매 필요)

**A. 도토리 수확 직원 — 다람쥐 (벌목꾼)**
- 외형: **청색 멜빵바지(작업복)** 입은 다람쥐, **도끼** 들고 있음.
- 모션: 정해진 나무 앞 한 곳에서 **도끼질 모션 반복**, 가끔 주변을 어슬렁(짧게 왕복) → 다시 도끼질. (경로탐색 불필요, 가벼움)
- 효과: **도토리를 시간당 n개 자동 인벤토리 적립**.

**B. 바나나 수확 직원 — 다람쥐 (운반꾼)**
- 외형: **조금 더 연한 색** 다람쥐, 청 멜빵바지, **리어카(손수레)** 끌고 다님. 리어카 안에 **바나나처럼 보이는 더미**가 쌓여 있음.
- 모션: 바나나 나무들 사이를 **천천히 순회(끌고 돌아다님)**, 나무에 닿을 때마다 수확 연출.
- 효과: **바나나를 시간당 n개 자동 적립**.

---

## 4. 아래 섬 — 🐟 물고기 & 🐚 조개 대량 생산 (아주 작은 섬 + 바다)

작은 섬 + 주변 바다를 활용. 좌/우 바다에 각각 시설.

### 4-1. 왼쪽 — 물고기 양식장
- 바다 위에 **배가 둥둥** 떠 있음(가벼운 위아래 흔들림 애니메이션).
- 직원: **밀짚모자 쓴 곰** — 배 위/물가에서 **낚시 모션**(낚싯대를 던지고 당기는 루프). 가끔 물고기가 "퍽!" 하고 잡히는 연출.
- 효과: **물고기 시간당 n개 자동 적립**.

### 4-2. 오른쪽 — 조개 수확장
- **해달 2형제**가 **물 위에 등을 대고 둥둥** 떠서 배 위에 조개를 올려놓고 있는 모습(해달 특유 포즈).
- 모션: 둥실둥실 흔들리며 조개를 **두드리는/까는 모션** 반복.
- 효과: **조개 시간당 n개 자동 적립**.

## 5. 직원/시설 시스템 (핵심 메커니즘)

### 5-1. 고용 & 레벨업
- 각 시설 직원은 **잠금 상태**로 시작 → **코인으로 고용**(첫 고용 비용).
- 고용 후 **레벨업(추가 코인)**으로 **시간당 생산량 증가**. (선택: 최대 레벨 상한)

### 5-2. 생산 공식 (제안값 — 튜닝 전 초안)
> "시간당 n개"를 부드럽게 하기 위해 **생산 주기 = 누적 시간**으로 처리.
> `생산량/초 = base × level` 형태. 화면엔 일정량 모일 때마다 1개씩 톡 적립.

| 자원 | 1차 고용 비용 | 레벨업 비용(예) | Lv1 생산 | 레벨당 증가 | 최대 Lv |
|---|---|---|---|---|---|
| 도토리 | 800 | ×1.7 | 1.5/초 | +1.2/초 | 8 |
| 바나나 | 1,200 | ×1.7 | 1.2/초 | +1.0/초 | 8 |
| 물고기 | 2,000 | ×1.7 | 0.9/초 | +0.8/초 | 8 |
| 조개 | 1,800 | ×1.7 | 1.0/초 | +0.9/초 | 8 |

- 값은 **`GameState`의 한 곳(레지스트리)**에 두어 밸런싱은 숫자만 수정.
- **상위 자원일수록 단가↑·생산속도↓** 로 균형.

### 5-3. 인벤토리 상한 / 흐름
- 생산 자원은 인벤토리에 쌓임 → **진열 알바**가 선반으로 → 손님 구매.
- (선택) **인벤토리 상한**을 둬서 너무 무한정 쌓이지 않게(생산이 상한에 닿으면 일시정지 = "창고 꽉 참" 표시). 후반 진열 알바 레벨/선반 슬롯과 균형.

### 5-4. (선택) 오프라인 생산
- 저장/불러오기 시스템이 이미 있으므로, **자리를 비운 시간만큼(상한 내) 생산 누적** 가능. "돌아오니 자원이 쌓여 있음" → idle 게임 손맛. (상한 예: 2시간치)

## 6. UI / 피드백

- **시설 고용 패널**: 하단 알바 패널과 통합 또는, 해당 시설을 **탭하면 그 시설 전용 고용/레벨업 팝업**(아이콘·현재 Lv·생산/초·다음 비용).
- **생산 인디케이터**: 각 시설 위에 "🌰 ×지금까지 N" 또는 자원 아이콘 + 시간당 수치 표시.
- **자원이 적립될 때** 작은 아이템이 인벤토리로 날아가는 연출(기존 `flyHarvest` 재사용).
- **창고 꽉 참** 상태면 시설 위에 경고 아이콘.

## 7. 아트 디렉션 (저폴리, 기존 톤 유지)

- 전부 **프로그래밍 메시(Box/Cylinder/Sphere/Cone)** + 토온 머티리얼, 외부 에셋 없음.
- **다람쥐 직원**: 기존 손님 다람쥐 베이스 재활용 + 멜빵바지(가슴판+끈 박스), 도끼(자루+머리), 리어카(바퀴 2 + 짐칸 + 바나나 더미).
- **곰**: 둥근 몸 + 밀짚모자(기존 알바 모자 응용) + 낚싯대(가는 실린더 + 줄).
- **해달 2형제**: 길쭉한 몸을 눕힌 포즈, 배 위 조개(작은 분홍 돔), 물 위 부유(상하 sin).
- **모션**: 전부 **제자리 루프 애니메이션**(도끼질=팔 회전, 낚시=낚싯대 끄덕, 부유=상하). 경로탐색 없음 → 성능 안전.

## 9. 밸런스 설계 방향
- **이 구역은 "후반 수입 엔진"**: 손님 수(스테이지당 +2)·다구매 손님과 맞물려, 시설 생산이 공급을 받쳐줘야 함.
- 권장 흐름: 시설 고용 → 생산 → 진열 알바 자동 진열 → 손님 대량 구매 → 그 코인으로 시설 레벨업 → 선순환.
- **체감 목표**: "직접 캐던 초반"에서 "공장처럼 돌아가는 후반"으로 자연스러운 전환.

## 10. 확장 가능 구조에 얹는 기술 설계 (구현 시 가이드)

> 새 항목 추가가 "설정/빌더 1개 등록"으로 끝나도록 — 기존 패턴 그대로.

1. **GameState**
   - `STAGES[5].unlocks`에 `'upper_island'`, `'lower_island'` 추가(또는 신규 6단계로 분리).
   - **시설 레지스트리** 추가: `FACILITIES = { acornFarm:{item:'acorn', base, mul, maxLv, cost...}, bananaFarm, fishFarm, clamFarm }`.
   - 시설 레벨 상태(`this.facilities = {acornFarm:0, ...}`) + `facilityCost(id)`, `hireFacility(id)`, `facilityRate(id)` 메서드(알바 메서드와 동형).
2. **Island (피처 빌더)**
   - `_buildUpperIsland()` / `_buildLowerIsland()` + 다리 + 나무/배/시설 노드 등록.
   - `getReadyTargets`와 무관(자동 생산이라 노드 탭 불필요) — 대신 시설 위치만 노출.
3. **Worker / 신규 FacilityManager**
   - 시설 직원 캐릭터 빌더(`makeAxeSquirrel`, `makeCartSquirrel`, `makeFisherBear`, `makeOtters`) + 제자리 루프 애니메이션.
   - `update(dt)`: 각 시설 레벨>0이면 `acc += facilityRate(id)*dt`; `acc>=1`마다 `gameState.addToInventory(item)` + 적립 연출.
4. **UI**
   - 시설 고용/레벨업 패널(알바 패널 확장) — 데이터 주도로 `FACILITIES` 순회 렌더.
5. **main**
   - 업그레이드 시 `island.applyUnlocks` 자동 처리, 카메라 포커스(위/아래 섬 보이게 viewW/viewShift 또는 자유카메라 그대로).

## 11. 결정 필요 항목 (확정해 주시면 구현 착수)
1. **해금 시점**: 5단계 진입 즉시 위·아래 섬 둘 다 열림? 아니면 **6·7단계로 분리**(위→아래 순차)?
2. **수동 수확과 공존**: 기존 메인 섬의 직접 탭 수확은 그대로 두나요, 아니면 후반엔 시설 위주로?
3. **인벤토리 상한 / 오프라인 생산** 도입 여부.
4. **고용 UI**: 시설 탭→개별 팝업 vs 하단 패널에 한꺼번에.
5. **밸런스 숫자**(5-2 표) 이대로 시작할지, 더 빠르게/느리게.
