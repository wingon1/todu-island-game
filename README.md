# Todu Island Game

모바일 세로 화면에 맞춘 저폴리 3D 코지 아일랜드 타이쿤 게임입니다. 작은 섬에서 직접 자원을 수확하고, 선반에 상품을 진열하고, 동물 손님에게 판매하면서 상점을 5단계까지 확장하는 구조입니다.

게임은 Three.js와 Vite 기반으로 동작하며, 외부 3D/이미지/오디오 에셋 없이 모든 섬, 상점, 동물, 아이템, 사운드를 코드로 생성합니다.

## 실행 방법

```bash
npm i
npm run dev
```

빌드:

```bash
npm run build
```

프리뷰:

```bash
npm run preview
```

## 게임 컨셉

플레이어는 작은 무인도 상점을 운영합니다. 초반에는 도토리를 직접 수확해 작은 나무 가판대에서 판매하고, 이후 바나나, 물고기, 조개, 꽃, 꿀까지 상품군이 늘어납니다. 섬은 단계가 오를수록 우측 가든 섬, 위쪽 생산 섬, 아래쪽 작은 바다 생산 섬으로 확장됩니다.

핵심 흐름:

1. 자원 노드가 준비되면 탭해서 수확
2. 인벤토리에 들어온 아이템을 선반에 진열
3. 동물 손님이 원하는 상품을 구매
4. 코인을 모아 상점과 섬을 업그레이드
5. 알바와 생산 시설로 후반 자동화를 확장

## 스테이지 구성

| 단계 | 상점 | 해금 자원 | 손님 | 선반 | 주요 변화 |
|---|---|---|---|---:|---|
| 1 | Wooden Stall | 도토리 | 다람쥐 | 2 | 메인 섬, 오크 나무, 기본 판매 루프 |
| 2 | Fabric Tent | 도토리, 바나나 | 다람쥐, 원숭이 | 4 | 야자수와 바나나 수확 해금 |
| 3 | Thatch Store | 도토리, 바나나, 물고기, 조개 | 토끼, 고양이 추가 | 6 | 해변 아이템 자동 스폰, 벤치, 볏짚 상점 |
| 4 | Garden Stall | 꽃 추가 | 곰 추가 | 8 | 우측 가든 섬, 다리, 꽃밭, 통나무 상점 |
| 5 | Garden Café | 꿀 추가 | 여우 추가 | 10 | 분수, 벌집, 캠핑카, 피크닉 테이블, 위/아래 생산 섬, 시설 시스템 |

## 섬 확장 내용

### 메인 섬

- 비정형 타원 형태의 잔디 코어와 밝은 모래 해변으로 구성
- 오크 나무 3개, 야자수 2개, 해변 스폰 지점 배치
- 스테이지 5에서 듬성듬성한 돌길이 추가됨
- 고객과 수확 알바가 상점 앞 동선을 기준으로 이동

### 우측 가든 섬

- 스테이지 4에서 다리와 함께 해금
- 수확 가능한 꽃 노드 2개
- 파스텔 분홍꽃, 노랑꽃, 하늘색 꽃 장식이 잔디 위에 배치
- 스테이지 5에서 연못 대신 분수와 벌집이 추가됨
- 꿀은 최종 단계의 핵심 판매 자원

### 위쪽 생산 섬

- 스테이지 5에서 해금
- 도토리 농장과 바나나 농장 시설이 배치됨
- 중앙에는 오크 나무, 바깥쪽에는 야자수가 배치됨
- 시설 고용 후 자동 생산이 가능

### 아래쪽 생산 섬

- 스테이지 5에서 해금
- 작고 아담한 섬과 바다 시설로 구성
- 왼쪽에는 물고기 양식장, 오른쪽에는 조개 수확장이 배치됨
- 시설 직원 캐릭터가 제자리 루프 애니메이션으로 생산감을 보여줌

## 자원

| 자원 | 최초 해금 | 획득 방식 | 판매 가치 |
|---|---:|---|---:|
| 도토리 | 1 | 오크 나무 수확 | 6 |
| 바나나 | 2 | 야자수 수확 | 9 |
| 물고기 | 3 | 해변 스폰 수확, 시설 생산 | 14 |
| 조개 | 3 | 해변 스폰 수확, 시설 생산 | 11 |
| 꽃 | 4 | 우측 가든 섬 꽃 노드 수확 | 18 |
| 꿀 | 5 | 벌집 수확 | 24 |

테스트 모드가 켜져 있으면 판매 코인은 5배로 적용됩니다.

## 손님 시스템

동물 손님은 일정 간격으로 섬에 들어와 상점 앞까지 이동합니다.

- 스테이지가 오를수록 손님 종류와 동시 방문 가능 수가 증가
- 손님은 현재 스테이지에서 판매 가능한 아이템 중 하나 이상을 원함
- 말풍선에는 원하는 아이템이 표시됨
- 선반에 원하는 상품이 있으면 구매
- 6초 내 구매하지 못하면 실망 효과와 함께 퇴장
- 최종 스테이지에서 꿀 첫 판매가 발생하면 목표 달성 조건이 충족됨

## 진열 시스템

선반에는 스테이지별 슬롯 수만큼 아이템을 올릴 수 있습니다.

- `Stock Shelf` 버튼으로 수동 진열 가능
- 진열 알바 고용 시 자동 진열
- 진열 로직은 한 종류만 몰리지 않도록 선반 위 아이템 수가 적은 자원을 우선 선택
- 인벤토리에 실제 보유한 자원만 선반에 올라감

## 알바 시스템

현재 알바는 2종입니다.

| 알바 | 역할 | 레벨 구조 |
|---|---|---|
| 수확 알바 | 준비된 자원 노드를 순서대로 수확 | Lv1 도토리, Lv2 바나나, Lv3 해변, Lv4 꽃, Lv5 꿀, Lv6 속도 증가 |
| 진열 알바 | 인벤토리의 상품을 선반에 자동 진열 | Lv1 자동 진열, Lv2~3 보충 속도 증가, Lv4~5 묶음 진열 |

수확 알바는 가능한 자원을 `도토리 -> 바나나 -> 물고기/조개 -> 꽃 -> 꿀` 흐름으로 순환하며, 현재 순번의 자원이 준비되지 않았으면 대기합니다. 섬 사이를 이동할 때는 다리를 경유하도록 설계되어 있습니다.

선반은 무한히 늘리지 않고, 상점 팝업에서 진열 설비를 구매해 정돈할 수 있습니다. Stage 4에서는 `바구니 진열`, Stage 5에서는 `상자 진열`을 구매할 수 있으며, 실제 재고 수량은 유지하되 화면에는 상품별 대표 오브젝트만 표시해 후반 자동화가 커져도 상점이 지저분해지지 않도록 설계했습니다.

## 생산 시설 시스템

스테이지 5부터 후반 자동 생산용 시설을 고용할 수 있습니다. 시설은 직접 수확 노드가 아니라, 코인을 지불해 고용/레벨업하면 일정 속도로 인벤토리에 자원을 자동 추가합니다.

| 시설 | 생산 자원 | 기본 비용 | Lv1 생산량 | 레벨당 증가 | 최대 Lv |
|---|---|---:|---:|---:|---:|
| 도토리 농장 | 도토리 | 800 | 1.5/초 | +1.2/초 | 8 |
| 바나나 농장 | 바나나 | 1200 | 1.2/초 | +1.0/초 | 8 |
| 물고기 양식장 | 물고기 | 2000 | 0.9/초 | +0.8/초 | 8 |
| 조개 수확장 | 조개 | 1800 | 1.0/초 | +0.9/초 | 8 |

시설별 연출:

- 도토리 농장: 도끼를 든 다람쥐 직원
- 바나나 농장: 리어카를 끄는 다람쥐 직원
- 물고기 양식장: 밀짚모자를 쓴 곰 낚시꾼
- 조개 수확장: 물 위에 떠 있는 해달 2형제

## UI 구성

- 상단 HUD: 단계 표시, 코인, 현재 스테이지에서 사용 가능한 인벤토리 아이콘
- 하단 바: 업그레이드, 진열, 알바 고용 버튼
- 시설 팝업: 시설 탭 시 레벨, 생산량, 고용/레벨업 비용 표시
- 시작 게이트: 첫 입력으로 오디오 컨텍스트 초기화
- 승리 오버레이: 최종 판매 후 총 코인, 손님 수, 플레이 시간 표시

## 저장

로컬 자동 저장을 사용합니다.

- 저장 위치: 브라우저 `localStorage`
- 저장 키: `cozy-island-tycoon-save-v1`
- 저장 대상: 스테이지, 코인, 인벤토리, 선반, 누적 코인, 손님 수, 알바 레벨, 시설 레벨, 경과 시간
- 페이지 숨김/종료 시점과 주기적 자동 저장을 함께 사용

## 테스트 모드

테스트 모드는 `src/GameState.js`에서 제어합니다.

```js
export const TEST_MODE = true;
```

`true`일 때:

- 판매 코인 5배
- 테스트 돈 추가 버튼 표시
- 다음 스테이지 진행 테스트가 쉬워짐

출시/일반 플레이 상태로 둘 때는 `false`로 변경합니다.

## 확장 설계

스테이지와 기능은 레지스트리 기반으로 확장할 수 있습니다.

### 새 자원 추가

1. `GameState.js`의 `ITEMS`에 자원 추가
2. `toon.js`에 3D 아이템 메시 추가
3. `UI.js`에 HUD 아이콘 추가
4. 필요한 수확 노드나 시설을 `Island.js`에 추가

### 새 스테이지 추가

1. `GameState.js`의 `STAGES`에 단계 추가
2. `Store.js`의 stage builder에 상점 모델 추가
3. `Island.js`의 feature builder에 해금 기능 추가
4. 필요하면 손님/시설/알바 규칙을 레지스트리에 추가

### 새 시설 추가

1. `GameState.js`의 `FACILITIES`에 시설 데이터 추가
2. `Island.js`에 시설 위치와 hit target 등록
3. `Facility.js`에 직원 모델과 애니메이션 추가
4. UI는 `FACILITIES` 데이터를 읽어 팝업 내용을 갱신

## 아트/사운드 방향

- 모든 모델은 `BoxGeometry`, `CylinderGeometry`, `SphereGeometry`, `ConeGeometry` 등 기본 지오메트리로 구성
- 토온/저폴리 느낌을 유지하되, 동글동글하고 아기자기한 실루엣을 우선
- 외부 이미지, glTF, mp3, wav 없이 자체 생성
- 배경음은 Web Audio API로 합성한 부드러운 코드 진행과 낮은 파도 노이즈를 사용
